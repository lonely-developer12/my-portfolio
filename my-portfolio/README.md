# Uvod u Create React App

Ovaj projekat je pokrenut pomoću [Create React App](https://github.com/facebook/create-react-app).

## Dostupni Skripti

U direktorijumu projekta možete pokrenuti:

### `npm start`

Pokreće aplikaciju u razvojnog režimu.\
Otvorite [http://localhost:3000](http://localhost:3000) da je vidite u vašem pretraživaču.

Stranica će se ponovo učitati kada napravite promjene.\
Također možete vidjeti sve greške u konzoli.

### `npm test`

Pokreće testni runner u interaktivnom režimu.\
Pogledajte odjeljak o [pokretanju testova](https://facebook.github.io/create-react-app/docs/running-tests) za više informacija.

### `npm run build`

Gradi aplikaciju za produkciju u `build` folder.\
Ispravno pakira React u produkcijskom režimu i optimizuje gradnju za najbolju performansu.

Gradnja je minificirana i imena datoteka uključuju hashove.\
Vaša aplikacija je spremna za implementaciju!

Pogledajte odjeljak o [implementaciji](https://facebook.github.io/create-react-app/docs/deployment) za više informacija.

### `npm run eject`

**Napomena: ovo je operacija koja se ne može vratiti. Kada `eject`, ne možete se vratiti!**

Ako niste zadovoljni alatima za gradnju i izborom konfiguracije, možete `eject` u bilo kojem trenutku. Ova komanda će ukloniti jedinstvenu zavisnost za gradnju iz vašeg projekta.

Umjesto toga, kopiraće sve konfiguracione datoteke i transitive zavisnosti (webpack, Babel, ESLint, itd.) direktno u vaš projekat tako da imate potpunu kontrolu nad njima. Sve komande osim `eject` će i dalje raditi, ali će se odnositi na kopirane skripte tako da ih možete prilagoditi. U ovom trenutku ste sami.

Nikada ne morate koristiti `eject`. Kurirana funkcionalnost je pogodna za male i srednje implementacije, i ne biste trebali osjećati obavezu da koristite ovu funkciju. Međutim, razumijemo da ovaj alat ne bi bio koristan ako ne biste mogli prilagoditi kada ste spremni za to.
